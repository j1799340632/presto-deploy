1. Language and page title

In frontend/index.html I set <html lang="en"> and <title>Presto</title>. Screen readers read the page in the right language and announce the right title.

2. Semantic HTML

Every page uses real landmark tags, not just <div>s:
- <main> wraps the page content on every screen.
- <header> holds the top bar on the dashboard and edit pages.
- <nav> wraps the Login / Register links on the landing page.
- <section> groups related content.
This lets screen-reader users jump between regions.

3. Forms have real labels

Every input has a <label htmlFor="...">. 
Clicking the label focuses the input, and screen readers read the label out loud.
I also set autoComplete on every auth field so password managers work:
- Register name -> "name"
- Register / login email -> "email"
- Register password -> "new-password"
- Login password -> "current-password"

4. Alt text on images

Every <img> on a slide uses an alt attribute. The Add Image modal has its own "Alt Text (description)" field so the user is reminded to fill it in.
The sparkle emoji 🪄 in the logo is marked as aria-hidden="true" so screen readers don't read it as "sparkles".

5. Buttons with only an icon have aria-label

If a button has just an icon, I added aria-label so screen readers know what it does. Examples:
- aria-label="Previous slide" on the ◀ arrow
- aria-label="Next slide" on the ▶ arrow
- aria-label="Edit presentation title" on the ✎ button
- aria-label="Close error message" on every ✕ button
- aria-label="Version history" on the 🕐 button
- aria-label="Open slide control panel" on the ☰ Slides button

6. Error messages as alerts

Every error banner uses role="alert". As soon as React shows the banner, the screen reader reads the message.
Every banner also has a ✕ button to close it.

7. Keyboard support

You can use Presto without a mouse:
- Tab moves through buttons and inputs in order.
- Enter submits every form (login, register, new presentation, edit title, and all the element modals).
- On the edit page, Left/Right arrow keys switch slides.
- Escape closes any open modal.

8. Visible focus

When you Tab to a button or input, you can see where the focus is. The main buttons use:
Inputs change their border colour to purple when focused.

9. Good colour contrast

I used a dark theme with high-contrast text.

10. Text size

All font sizes use rem or em, not fixed px. The landing page title uses clamp(2.5rem, 6vw, 4rem) so the text scales with the screen. Users can also zoom the browser to 200% without breaking the layout.
